Python Basics

⏱ 20 minutes

Python is one of the most widely used programming languages in data science. It is clear to read, easy to learn, and has a rich ecosystem of tools for data analysis and machine learning. In this session we cover just enough Python to follow the rest of the course — pandas and scikit-learn do the heavy lifting from here on.

Variables and Data Types

A variable is a labelled container for a value. Think of it as a named test tube. Python works out the type automatically — generally you need to declare it.

# A simple patient record
patient_age = 67          # integer (int)
bmi = 28.4                # decimal number (float)
diagnosis = "Type 2 DM"   # text (str)
readmitted = True         # yes/no flag (bool)

print(patient_age)
print(type(patient_age))
67
<class 'int'>
Python is case-sensitive

patient_age and Patient_Age are different variables. By convention, Python variable names use lower_snake_case.

Operators

Operators are symbols that tell the interpreter to perform operations on values and variables—common families include arithmetic and Boolean (comparison and logical) operators.

Arithmetic Operators Meaning
+ Add
- Subtract
* Multiply
/ Divide
** Power (e.g., 2**3 = 8)
Comparison Operators Meaning
== Equal to
!= Not equal
< Less than
<= Less than or equal
> Greater than
>= Greater than or equal
Logical Operators Meaning
and True if both are True
or True if at least one is True
not Flips True ↔︎ False 
# Calculating BMI from weight and height
weight_kg = 82
height_m = 1.75

bmi = weight_kg / height_m ** 2     # ** means "to the power of"
print(f"BMI: {bmi:.1f}")            # :.1f formats to 1 decimal place
BMI: 26.8
# Using the BMI from above with Boolean operators
is_overweight = bmi >= 25          # comparison -> True or False
is_obese = bmi >= 30

needs_follow_up = is_overweight and not is_obese  # logical combination

print("Overweight?", is_overweight)
print("Obese?", is_obese)
print("Needs follow-up (overweight but not obese)?", needs_follow_up)
Overweight? True
Obese? False
Needs follow-up (overweight but not obese)? True

String formatting with f-strings

The f"..." syntax (f-string) lets you embed variable values directly into text — very useful for generating readable output:

name = "Smith"
age = 71
ward = "Cardiology"

print(f"Patient {name}, age {age}, admitted to {ward}")
Patient Smith, age 71, admitted to Cardiology

Lists

A list stores multiple values in order. Lists use square brackets [] and can hold any mix of types.

# HbA1c readings over 6 months (mmol/mol)
hba1c_readings = [52, 64, 71, 48, 89, 55]

print(hba1c_readings[0])    # First element (index 0)
print(hba1c_readings[-1])   # Last element
print(len(hba1c_readings))  # Number of elements
52
55
6

Conditionals (if / else)

Conditionals let your code make decisions — essential for flagging abnormal values.

hba1c = 71   # mmol/mol

if hba1c >= 48 and hba1c < 58:
    print("Pre-diabetes range (NICE criteria)")
elif hba1c >= 58:
    print("Diabetes range — review management")
else:
    print("Within normal limits")
Diabetes range — review management

The comparison operators are: == (equals), != (not equals), <, >, <=, >=. Combine conditions with and, or, not.

For Loops

A for loop repeats an action for every item in a list (or other collection).

# Flag each patient's eGFR reading
egfr_values = [85, 42, 61, 28, 90, 55]

for egfr in egfr_values:
    if egfr < 30:
        stage = "Stage 4 CKD"
    elif egfr < 60:
        stage = "Stage 3 CKD"
    else:
        stage = "Normal / Stage 1–2"
    print(f"eGFR {egfr}: {stage}")
eGFR 85: Normal / Stage 1–2
eGFR 42: Stage 3 CKD
eGFR 61: Normal / Stage 1–2
eGFR 28: Stage 4 CKD
eGFR 90: Normal / Stage 1–2
eGFR 55: Stage 3 CKD

Functions

A function packages reusable code into a named block. You define it once and call it many times — just like a clinical algorithm or scoring system.

def charlson_risk_category(score):
    """Classify a Charlson Comorbidity Index score into risk tier.
    
    Args:
        score (int): Charlson Comorbidity Index (0–37)
    
    Returns:
        str: Risk category label
    """
    if score == 0:
        return "Low"
    elif score <= 2:
        return "Moderate"
    elif score <= 4:
        return "High"
    else:
        return "Very High"

# Test it
for score in [0, 1, 3, 6]:
    category = charlson_risk_category(score)
    print(f"Charlson {score}: {category}")
Charlson 0: Low
Charlson 1: Moderate
Charlson 3: High
Charlson 6: Very High

Importing Libraries

Python’s real power comes from its libraries. We import them at the top of a notebook:

import pandas as pd          # Data manipulation — we use this alias throughout
import numpy as np           # Numerical operations
import matplotlib.pyplot as plt   # Plotting
import seaborn as sns        # Statistical plots (built on matplotlib)

The as pd part creates a short alias so we write pd.read_csv(...) instead of pandas.read_csv(...).
