Filtering & Missing Data

⏱ 15 minutes

Filtering Rows

Filtering in pandas works by passing a boolean condition inside square brackets. The condition produces a Series of True/False values — one per row — and pandas keeps only the True rows.

Single condition

# All patients aged over 65
elderly = df[df["age"] > 65]
print(f"Patients over 65: {len(elderly)}")
Patients over 65: 2036
Patients over 65: 2341

Multiple conditions

Use & (AND) and | (OR) to combine conditions. Each condition must be wrapped in parentheses:

# Elderly patients with Type 2 diabetes (E11)
elderly_dm = df[(df["age"] > 65) & (df["primary_diagnosis"] == "E11")]
print(f"Elderly T2DM patients: {len(elderly_dm)}")
print(elderly_dm[["age", "hba1c", "charlson_index"]].head())

# Patients with a missing HbA1c
missing_hba1c = df[df["hba1c"].isnull()]

Handling Missing Data

Missing data is endemic in medical records. Understanding why data is missing matters as much as knowing how to handle it technically.

The three types of missingness

Type What it means Example
MCAR — Missing Completely At Random Missingness unrelated to any variable Lab machine malfunction on a random Tuesday
MAR — Missing At Random Missingness related to observed variables HbA1c only ordered for diabetic patients
MNAR — Missing Not At Random Missingness related to the missing value itself Very sick patients not surviving to have tests done

MNAR is the most dangerous type — simply imputing or dropping these rows introduces bias.

Strategy 1: Drop rows with missing values

# Drop any row with a missing value in critical columns
df_dropped = df.dropna(subset=["hba1c", "egfr"])
print(f"Rows after dropping missing hba1c/egfr: {len(df_dropped)}")
print(f"Rows removed: {len(df) - len(df_dropped)}")
Rows after dropping missing hba1c/egfr: 3821
Rows removed: 1181
Use dropna() with caution

Dropping 24% of your data introduces selection bias — the remaining patients are those who had both tests ordered, which is not random. In a real clinical study, this would require formal justification in your methods section.

Strategy 2: Impute with the median

For skewed clinical variables (e.g. length of stay, HbA1c), the median is a better imputation value than the mean because it is robust to outliers:

# Impute missing BMI with the median
bmi_median = df["bmi"].median()
print(f"Median BMI: {bmi_median:.1f}")

df["bmi"] = df["bmi"].fillna(bmi_median)
print(f"Remaining BMI missing: {df['bmi'].isnull().sum()}")
Median BMI: 28.1
Remaining BMI missing: 0
Median BMI: 27.8
Remaining BMI missing: 0

For categorical variables, impute with the mode (most common value):

# Impute missing ward with the most common ward
ward_mode = df["discharge_ward"].mode()[0]
df["discharge_ward"] = df["discharge_ward"].fillna(ward_mode)

Strategy 3: Indicator column (recommended for MNAR)

Create a new column flagging whether the value was originally missing, then impute. This preserves the information that the value was absent:

# For HbA1c: flag missingness before imputing
df["hba1c_missing"] = df["hba1c"].isnull().astype(int)

# Then impute with median
df["hba1c"] = df["hba1c"].fillna(df["hba1c"].median())

Important points

The indicator column strategy is best practice for clinical machine learning when the reason for missingness is clinically meaningful. It is used in published NHS readmission models and tools like QRISK3.
